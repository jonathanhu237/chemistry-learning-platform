# 学生端产品学习模型

本文约束绿色学生 H5 的五栏信息架构与学习闭环。学生端是 H5 网站，不是微信原生小程序。

## 核心判断

产品不是闯关游戏，也不是独立视频库。更准确的定位是：

```text
以实验视频为入口、以教材目录为地图、以 Atom 和测评形成闭环的实验学习系统。
```

学生的主要路径是：

```text
看到实验现象
  -> 进入其教材点位
  -> 理解章节、原理和方程式
  -> 用 Atom 追问
  -> 用测评确认掌握程度
  -> 回到薄弱目录或点位复习
```

## 五个根入口

```text
首页 Home       = 有限的实验视频发现流
学习 Learn      = 教材目录与元素学习地图
Atom            = 带学习上下文的智能助教
测评 Assessment = 智能基线、组卷与薄弱点诊断
我的 Profile    = 收藏、报告、反馈与账号入口
```

五栏和当前绿色视觉系统是 canonical 产品边界。二级页面可以被多个根入口复用，但不能增加一个竞争性的第六个根产品。

## 首页：发现后回到点位

首页保留横屏实验视频流，适合现有 16:9 素材：

```text
静音封面/预览
实验标题
教材路径
现象、方程式或试剂摘要
明确的教师推荐标记（仅显式推荐时显示）
进入点位
```

交互约束：

- 主视区内只激活一个静音预览；离开主视区即暂停。
- 分页是有限且确定的，学生明确点击继续加载，不制造无限沉浸流。
- 首页内置聚焦搜索；默认流和搜索使用同一个 PostgreSQL 目录/视频 read model。
- 搜索只返回已发布且有可播放正式媒体的点位，游标必须绑定当前查询。
- 教师显式推荐排在普通目录内容之前；未标记内容不得伪装为推荐。
- 点击卡片进入其真实教材点位，而不是进入另一个视频库信息架构。
- 收藏是唯一持久化的视频保存动作，并在“我的收藏”中可撤销、可继续打开。

首页不包含独立视频库、现象话题轨、稍后再看、点赞、不感兴趣、分享、伪社交反馈等外围能力。没有后端持久化语义的按钮不应出现。

## 学习：目录、元素与 3D 模型

学习页是中立的内容地图，回答“这个实验在教材体系里的哪里”。真实层级是可变深度目录树：

```text
chapter
  -> directory
    -> directory
      -> point
```

目录可以按章节、周期表区域、元素族和元素进入，但所有入口最终解析到同一批发布目录节点和实验点位。学习页不把目录伪装成固定闯关路线。

元素详情保留当前 3D 原子/轨道学习体验。3D 模型是理解元素结构的学习能力，不是旧视觉风格或另一个应用入口。

点位详情是学生端的核心复用页：

```text
视频 + 实验现象 + 原理/方程式 + 用品/安全 + 相关点位 + Atom + 测评关联
```

首页、学习、收藏、相关推荐和测评回流都应进入这个共同 owner。

## Atom：带证据与上下文的助教

Atom 不是无上下文聊天玩具。来自点位、目录或测评报告的提问应携带可用的：

```text
实验标题
教材目录路径
元素、试剂、现象
结构化反应方程式
实验目的、原理和安全信息
测评薄弱点
```

检索证据来自当前发布教材语料的一套 RAG 投影。教材在教师端在线上传，经原文提取或 MinerU OCR、分块、Embedding、索引和发布后进入该语料；Rerank 决定最终证据顺序。OCR、Embedding 和 Rerank 的服务/模型由管理员在教师后台配置，不写死在学生端。

Atom 的浏览器聊天历史属于当前设备本地记录，不能文案承诺跨设备同步。回答需要区分证据、推断和不可用状态；RAG 不可用时不得伪造教材引用。

## 测评：一个基线，两种入口

首次学生完成强制密码修改后，只进入一套智能基线逻辑：没有已完成 `smart` 测评记录时，第一次智能组卷就是基线。系统不再叠加另一套 active pretest 门槛。

测评页保留两个入口：

### 智能组卷

按当前策略、掌握度和可用题目自动组卷。开放会话优先恢复，提交具备并发保护和幂等报告行为。多空题按空位顺序作答，并允许每个空位配置可接受的同义答案。

### 自主测评

学生搜索并选择：

```text
章节 -> 目录 -> 点位
```

上级范围会展开到其中可用的已发布叶子点位，并按稳定顺序去重。每个点位只能选择 1、2 或 3 题；题量不足时按真实可用题目返回，不用重复题目伪造配额。随机练习和“全部范围”不是额外入口。

测评结果更新 BKT/掌握度并生成报告，再把学生带回真实目录节点、点位视频或 Atom，而不是重建另一套目录。

历史 pretest 数据和报告标签仍可只读展示，保证既有学生记录可解释；学生端没有启动或提交新 pretest 的入口。

## 我的：持久记录而非功能仓库

“我的”负责：

- 账号与退出；
- 持久化实验视频收藏；
- 智能、自主、点位、后测以及历史 pretest 报告；
- 可用时提交课程、内容或系统反馈。

没有真实持久化 owner 的“看过”“稍后再看”或社交统计不应写进产品承诺。

## 三类学习流

```text
发现型
首页视频 -> 点位详情 -> 目录/Atom/测评

任务型
学习目录或元素 -> 3D 原子/轨道或实验点位 -> 视频与解释 -> Atom

诊断型
智能/自主测评 -> 掌握度与报告 -> 薄弱目录/点位 -> 再测评
```

三条路径共享目录、canonical point、媒体、教材证据和测评身份，不复制内容模型。

## 防止被误读为娱乐流

风险来自短视频交互形态，而不是实验视频本身。产品必须保持：

- 目录路径和化学信息可见；
- 预览默认静音且一次只播放主视区内容；
- 每条内容可进入点位；
- 显式推荐可追溯到教师配置；
- 分页有限，不以无限下滑为目标；
- 详情回到原理、方程式、安全、Atom 与测评；
- 收藏是学习回访动作，不包装成社交互动。

## 产品原则

- 用视频吸引学生进入真实实验点位。
- 用目录说明内容在哪里。
- 用 3D 原子/轨道帮助理解元素结构。
- 用 Atom 结合已发布教材证据解释为什么。
- 用一个智能基线和可选的层级自主测评判断会不会。
- 用报告和掌握度把学生带回薄弱点。
- 一个能力只有一个 canonical owner；不为同一内容维护两套目录、视频搜索或向量系统。
