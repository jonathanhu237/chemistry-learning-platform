# 在线教材 RAG 摄取

## Goal

教师或管理员可以在后台上传教材 PDF，由服务端在线完成解析、必要时 OCR、结构化、分块、Embedding、Elasticsearch 索引、发布和下游证据刷新，不再依赖开发者预先生成 JSONL、Embedding 文件或手工执行离线脚本。

在线表示用户从后台发起并查看任务状态；耗时步骤由服务端异步 worker 执行，而不是在 HTTP 上传请求或用户电脑浏览器中执行。

首版以《无机化学（下册）（第二版）》和《无机化学实验（第四版）》为主要且权威的教材语料，优先保证这两本教材能够完整在线摄取、检索和引用。架构仍按多教材和版本化设计，但首版验收不追求覆盖任意版式的 PDF。

## Background

- 当前教材语料来自固定的预处理 JSONL：`scripts/import_canonical_evidence.py` 将教材和 chunk 导入 PostgreSQL，`scripts/index_textbook_rag_chunks.py` 再调用 Embedding 并写入 Elasticsearch。
- 当前后台只提供教材 RAG 配置、健康状态和检索测试，没有教材上传、解析、任务进度、预览或发布 API。
- PostgreSQL 已有 `source_documents`、`source_chunks` 和历史 `chunk_embeddings` 表；现行教材检索使用 Elasticsearch 的关键词字段和 `dense_vector`。
- 项目已依赖 PyMuPDF、`pymupdf4llm`、`pdfplumber` 和 `pypdf`，适合进行原生 PDF 文本抽取。
- 当前开发机为 Apple M4、24GB 内存，普通 PDF 文本抽取能力充足。本机 Tesseract 仅安装英文、方向检测和数字模型，没有简体中文模型；本机 OCR 不应成为生产链路的必要条件。
- 已检查两份首版 PDF 样本：
  - 《无机化学（下册）（第二版）》共 299 页，全部页面有可提取文本，289 页包含至少 200 个非空白字符；部分封面文字和复杂公式存在抽取顺序或字符误差，137 页包含图片。
  - 《无机化学实验（第四版）》共 240 页，其中 239 页为扫描图片且基本没有文本层，只有末页含少量文本；将其纳入首版必须使用支持中文的 OCR。
- 中大校内 AIGW 已提供模型别名 `mineru`。网关公开模型信息将其标注为 `MinerU2.5-Pro-2604-1.2B`、`本地`、`网络中心`，实际无教材合成页调用的响应模型字段也返回 `mineru2.5-pro-2604-1.2b`，因此版本已经确认，不是旧版 MinerU 2.0/2.5 baseline。
- 校内 `mineru` 是 OpenAI-compatible 的页图 VLM 推理端点，不是接收整本 PDF 并直接返回 Markdown/JSON/Zip 的 MinerU 精准解析云 API。后台仍需负责 PDF 渲染、逐页调度、两阶段版面/内容识别、跨页整理和结果持久化；这些工作由服务端 worker 在线执行，不依赖用户电脑或本地 GPU。
- 使用官方 `mineru-vl-utils` 对无敏感合成页实测：版面类型和坐标基本正确、普通文本识别正确，但网关移除了官方客户端解析所需的特殊 token，只返回简化的 `x1 y1 x2 y2 type` 行；官方客户端因此会把布局解析为空。首版必须增加受测试的兼容解析层，不能仅凭模型名直接上线。
- 经用户明确许可，已将两本教材各 7 页、共 14 个代表页发送到校内 MinerU；没有上传整本教材。14/14 页请求成功，端到端平均约 3.27 秒/页；章节标题、正文、实验步骤、页码、图题和图注整体可读，13 个独立公式块均返回 LaTeX，9 个表格块均返回非空文字。第一本书的 7 页结果与原生文本层做归一化顺序相似度 sanity check，平均约 0.915；该数值不是人工标注准确率。
- 代表页验证同时确认两项适配要求：AIGW 会去掉布局 special token；表格识别也会丢失官方 HTML/cell token，只剩行文本和 LaTeX，stock `mineru-vl-utils` 后处理会进一步把这些有效表格文字清空。适配器必须保留原始表格文字、在必要时以 Text Recognition 降级，并对无法恢复单元格边界的扫描表格标记质量问题。
- 百度“文档解析（PaddleOCR-VL）”调整为校内 MinerU 不通过质量或稳定性门禁时的外部 fallback。校内网关的调用日志、数据留存、并发和 SLA 仍需向网络中心确认。
- 两本教材与现有离线 canonical source documents 对应。在线版本发布后必须替代相同教材的离线版本进入检索，不能产生新旧版本重复召回；旧版本保留为可回滚事实数据，确认稳定后再清理。

## Requirements

### R1. 上传与原文件管理

- 后台支持上传 PDF，上传接口完成格式、大小和 MIME 校验后立即返回教材记录与处理任务，不等待整本教材处理完成。
- 原文件必须持久化保存，并记录 SHA-256、文件名、大小、上传人和上传时间。
- 重复文件可被识别；重新上传同一本教材时创建新版本，不静默覆盖已发布版本。

### R2. 在线异步处理

- 通过数据库任务表和独立 worker 执行解析、OCR、分块、Embedding、索引和发布准备。
- 任务至少暴露 `uploaded`、`extracting`、`awaiting_ocr`、`ocr`、`structuring`、`chunking`、`embedding`、`indexing`、`review_ready`、`ready`、`failed`、`cancelled` 状态，以及进度、错误、重试次数和时间戳。
- 处理必须幂等，可安全重试、取消和从失败阶段重新执行；第一版不引入 Redis、Celery 或 RabbitMQ。

### R3. PDF 抽取与 OCR 降级

- 优先用 PyMuPDF/`pymupdf4llm` 提取 PDF 原生文本、页码和结构，避免对所有页面无条件 OCR。
- 按页执行质量检测；无文本、文本密度过低、乱码比例过高或结构明显异常的页面才进入 OCR。
- OCR/文档 VLM 通过可插拔 HTTP provider 调用，配置包括 Base URL、模型别名、认证信息、超时、并发、重试和单页/批量限制；认证差异只存在于 provider adapter，不要求管理员手工维护短期 token。
- OCR 服务不可用时任务必须明确停在 `awaiting_ocr` 或 `failed`，不得把空白或乱码内容发布到 RAG。
- OCR 请求和响应应支持页码、文本或 Markdown、文本块/坐标、置信度和可选图片资源；不得将 provider 特有字段泄漏到核心领域模型。
- 《无机化学实验（第四版）》的扫描页必须走 OCR HTTP provider；《无机化学（下册）（第二版）》默认使用文本层，仅对质量门禁失败的页面调用 OCR。
- 中大校内 AIGW `mineru` 是当前首选 provider，精确模型为 `MinerU2.5-Pro-2604-1.2B`。worker 将 PDF 页渲染为图片后，通过 OpenAI-compatible `/v1/chat/completions` 和官方 MinerU 两阶段提示完成版面检测及正文/公式/表格识别；provider adapter 负责把网关的无特殊 token 简化布局格式归一化为核心 block contract，并在 stock 后处理之前保留无 HTML 的表格原始文字。
- 百度智能云“文档解析（PaddleOCR-VL）”仅作为后续外部备选，不在 MVP 中实现。只有后续质量或稳定性证据推翻校内 MinerU 的当前门禁结论时，才对相同代表页重新评估百度；不在生产中长期维护两套并行解析结果。切换 provider 不应改变上传、任务和发布领域模型。
- 首次接入不得直接付费处理整本教材。先选取包含正文、标题、公式、表格、实验步骤、思考题和装置图题的代表页进行试用额度验证；达到质量门禁后再提交全书。

### R4. 教材结构化与化学内容保真

- 识别教材、章、节、小节、页码、正文、实验步骤、安全提示、表格、例题和图注等结构。
- 清理页眉页脚、孤立页码、重复目录和断行噪声。
- 尽量保留上下标、化学式、离子、电荷、反应方程式、元素和化合物；无法可靠识别的内容记录质量标记，供后台预览。
- 扫描页表格在首版允许降级为可检索的表题、行文字、数值、LaTeX 和页码，并标记 `table_structure_lost`；只要文字非空且通过质量门禁，缺少精确单元格/HTML 结构不阻塞文字 RAG 发布。
- 每个 chunk 必须携带 `document_id`、版本、教材名、章节路径、页码范围、内容类型、内容哈希和相邻/父级关系。
- 首版只索引文字内容。实验装置图和教材插图不做视觉语义理解，但应保留可提取/OCR 的图题、图号、邻近说明和原 PDF 页码。

### R5. 分块、质量门禁与预览

- 使用标题/段落/表格等结构感知分块，长度上限和 overlap 可配置，不以固定字符切割作为唯一策略。
- Chunk ID 基于文档版本、结构位置和内容哈希稳定生成，以支持重试与增量重建。
- 发布前检查空块、重复块、过短/过长块、页码覆盖、乱码、章节识别和抽取/OCR 质量。
- 后台可查看解析摘要、问题列表、章节树和 chunk 原文，并允许重新处理；第一版不要求在线富文本编辑 chunk。

### R6. 单一向量存储与索引发布

- PostgreSQL 保存教材、版本、chunk 正文、任务、质量报告和发布状态，作为事实源。
- Elasticsearch 是唯一向量存储，同时承载 BM25/IK 关键词索引、metadata filter 和 `dense_vector`。
- 新在线流程不得写入 PostgreSQL `chunk_embeddings`；确认无消费者后另行迁移删除该历史表。
- Embedding 按批执行并记录模型、维度和配置指纹；相同内容和相同配置可跳过重复计算。
- 索引写入必须校验 chunk 数量、失败项、Embedding 维度和索引元数据。未完全通过的教材版本不得被检索端视为已发布。

### R7. 发布、替换与删除

- 只有通过质量门禁和索引校验的版本才能发布；发布后检索端才可见。
- 新版本发布时旧版本可原子停用，不能让同一本教材多个版本无意中同时召回。
- 停用或删除教材时同步移除或屏蔽 Elasticsearch 文档，并使引用这些 chunk 的点位证据状态变为过期。
- 原文件、数据库事实数据和 ES 派生索引的删除边界必须明确；默认停用可恢复，物理删除需显式操作和审计。

### R8. 下游 RAG 与可观测性

- 发布教材后，受影响的目录点位教材证据应标记为 stale，并可批量或按点位刷新。
- 检索继续使用关键词召回、向量召回、候选合并、Rerank、元数据过滤和带页码引用的证据输出。
- 后台显示每阶段耗时、页数、OCR 页数、chunk 数量、Embedding 成功/失败数、ES 写入结果和最终检索冒烟结果。
- 记录解析器、OCR provider、Chunk 配置、Embedding 模型和索引 schema 的版本，以支持复现与重建。

## Acceptance Criteria

- [ ] 管理员上传一份可提取文本的 PDF 后，API 立即返回 document/job ID，任务可异步完成并在后台展示逐阶段状态。
- [ ] 文本型 PDF 不调用 OCR；扫描页或低质量页自动进入 OCR provider，OCR 不可用时不会错误发布。
- [ ] 处理结果在 PostgreSQL 中形成带教材版本、章节路径和页码引用的 `source_documents`/`source_chunks` 事实数据。
- [ ] 后台可以预览章节、chunk 和质量问题，并能重试失败任务。
- [ ] Embedding 只写入 Elasticsearch；新流程不会新增 `chunk_embeddings` 行。
- [ ] 同一任务重复执行不会产生重复文档、重复 chunk 或孤儿 ES 文档。
- [ ] 索引不完整、模型维度不匹配或质量门禁失败时，教材版本不可发布。
- [ ] 发布后能通过教材名/章节过滤执行关键词和向量检索，并返回正确教材、章节、页码与 chunk 引用。
- [ ] 替换或停用教材版本后，旧版本不再被召回，并将相关点位证据标记为过期。
- [ ] 现有离线种子仍可作为迁移/回归数据使用，但运行期新增教材不要求生成或提交 JSONL 种子文件。
- [ ] 《无机化学（下册）（第二版）》能够主要通过原生文本抽取完成摄取，并能从复杂公式页面返回可追溯的页码引用和质量诊断。
- [ ] 《无机化学实验（第四版）》能够通过 HTTP OCR 完成扫描页文字摄取，随机抽检章节标题、实验目的、实验步骤、思考题和图题可读，且无文本层不会导致空 chunk 被发布。
- [x] 校内 MinerU 代表页验证已记录正文可读性、标题/阅读顺序、公式、表格、图题、页码映射、耗时和兼容问题：14/14 页成功，已通过首版文字 RAG 代表页门禁；扫描表格精确单元格结构不属于 MVP 验收范围。
- [ ] 校内 MinerU adapter 能保留 AIGW 的简化布局和纯文本表格结果，不被 stock 后处理清空；对表格结构丢失、疑似公式字符错误和容器/子块重叠产生可见质量标记。
- [ ] 两本在线教材发布后，相同的离线 canonical 版本不再参与召回；回滚后可以恢复旧版本，不需要重新导入种子。

## Out of Scope for MVP

- DOCX、EPUB 等非 PDF 格式。
- 对教材插图进行视觉问答、生成图片语义描述，或把实验装置图转换为结构化装置知识；首版仅保留图题和邻近文字。
- 自动无人工确认地修复低置信度化学方程式。
- 从扫描页恢复精确表格单元格、合并关系或 HTML/Markdown 表格结构；首版保留可检索文字和质量标记。
- 多机分布式队列、任务优先级调度和自动扩缩容。
- 在后台逐字编辑整本教材或提供完整 PDF 编辑器。
