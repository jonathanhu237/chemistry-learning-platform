## Confirmed structural evidence

- Branch merge base: `3096f82d6b282d6a637910960c3123a2c7963d20`.
- Planning-time ahead counts: `main` 11 unique commits, `legacy` 117 unique commits.
- Twelve legacy-only commits touch the promoted legacy student frontend, including forced password change, real video previews, recommendation behavior, baseline assessment, assessment assembly modes, multi-blank answers, and submit/error UX.
- Current student H5 source size is approximately 22,894 TS/TSX lines plus 11,000 CSS lines. Several components and style owners exceed 1,000 lines.
- Current production main JavaScript asset is approximately 1.65 MB (about 444 KB gzip); the pre-refinement old student frontend builds to approximately 255 KB (about 76 KB gzip). Feature scope differs, so this is a performance signal rather than a direct quality score.
- Current visible no-op or ephemeral controls include Home `not-interested`/`feedback` messaging and point-detail `liked` component state.
- Existing `docs/student-product-learning-model.md` makes video discovery a foundational Home responsibility, and the user has confirmed that the video feed remains in scope. The surrounding library/search/recommendation/action surface still requires product triage.
- The terminology is ambiguous in existing code: latest legacy names its Home feed `HomeVideoLibraryPage` and displays the heading “实验视频库”, while current `main` has both a Home feed and a distinct `/video-library` route. Product decisions must distinguish the retained Home feed from the optional standalone library/search surface.
- Latest legacy has no student 3D atom/orbital viewer and no favorites capability. Any retained version of those features must come from current `main`, even if assessment behavior comes from legacy.
- Latest legacy tests explicitly assert that “随机练习” and “全部范围” are absent. Its final custom-assessment contract accepts catalog `scope_node_ids` plus `questions_per_point`, expands a selected chapter/directory to published leaf points, and validates that every chosen scope exists and has usable questions.
- A read-only `git merge-tree` preview reports 21 explicit conflicts, but its auto-merged tree switches both canonical frontend entrypoints to `LegacyStudentApp` / `LegacyTeacherApp` and deletes many retained current components without conflict markers. Implementation therefore needs a semantic baseline guard, not only conflict-marker review.
- The current Home feed already owns a queryable published point/media read model and cursor contract. Adding normalized `q` filtering and recommendation-aware deterministic ordering there removes the need for a second student discovery index.
- Teacher catalog search imports transport/analyzer utilities from the current `video_library` package but uses its own index, mapping, state and configuration. Shared utilities must move to a neutral module before deleting the student projection; textbook RAG and teacher catalog search are not deletion targets.
